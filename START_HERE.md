   You'll configure these in GitHub's web interface (20 minutes):

   1. Branch Protection Rules
      Settings → Branches → Add rules for 'main' & 'develop'
      └─ Require PR reviews, status checks, up-to-date branches

   2. Merge Strategy
      Settings → General → Configure pull request merge options
      └─ Enable squash merge, disable merge commits

   3. Action Permissions  
      Settings → Actions → Enable read/write permissions
      └─ Allow workflows to create releases

   4. Dependabot Security
      Settings → Code security → Enable Dependabot alerts & updates
      └─ Auto-update dependencies, alert on vulnerabilities

   5. Repository Details
      Repo homepage → Add topics, set description, choose MIT license
      └─ Help users understand and find your project
